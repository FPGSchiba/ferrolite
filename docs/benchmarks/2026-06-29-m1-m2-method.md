# Ferrolite M1/M2 Benchmark Methodology

**Date:** 2026-06-29  
**Spec ref:** §8 (performance benchmarks)  
**Status:** method committed; head-to-head vs RawTherapee run by user on benchmark machine

---

## Overview

Two benchmark scenarios measure ferrolite's browsing performance against RawTherapee
on an identical machine and dataset:

| ID | Scenario | Measure |
|----|----------|---------|
| M1 | Cold folder-open → first thumbnails | Wall-clock time to index rows (M1a) and to commit the first N thumbnails (M1b); throughput in thumbnails/sec |
| M2 | Warm grid scroll smoothness | Visual observation of dropped frames while scrolling a fully-indexed folder |

Image quality is deliberately **not** compared — ferrolite generates 256px JPEG q85
preview thumbnails, which is sufficient for browsing.

---

## Dataset

- **Size:** ~2,000 24 MP RAW files (target; fewer files produce proportionally faster
  but valid numbers)
- **Formats:** mix of .CR2, .NEF, .ARW, .RW2, .RAF or similar
- **Source:** personal archive; files are referenced by a committed list (see below)
  but the actual files are **not** committed to this repo
- **File list:** `docs/benchmarks/dataset-file-list.txt` (committed; each line is a
  filename as it appears in the test folder — paths are relative to the folder root)

> **Note:** A single-file smoke run against `fixtures/raw/sample.rw2` is sufficient
> to verify the harness works on the developer machine (see §Baseline below).

## M1: Headless Harness

The `bench_browse` binary exercises the **real** ferrolite ingest pipeline — the same
`scan_raw_files` + `read_metadata` + `decode_preview` + `generate_thumbnail` +
`put_thumbnail` code path used in the interactive app — without any egui dependency.

### Commands

```bash
# Build release binary (recommended for benchmark runs):
cargo build -p ferrolite-app --bin bench_browse --release

# Run against a folder of RAW files, timing the first 100 thumbnails for M1b:
cargo run -p ferrolite-app --bin bench_browse --release -- <path-to-raw-folder> 100

# Example — full dataset (2000 files, time first 100 thumbnails):
cargo run -p ferrolite-app --bin bench_browse --release \
  -- /Volumes/Photos/benchmark-set-2000 100

# Example — smoke test with fixture file:
cargo run -p ferrolite-app --bin bench_browse -- fixtures/raw 1
```

### Output Fields

| Field | Definition |
|-------|-----------|
| `M1a rows-indexed-complete` | Wall-clock ms from start until all rows are upserted into the catalog |
| `M1b first-N-thumbnails-committed` | Wall-clock ms from the end of M1a until N thumbnails have been decoded, resized, and written to the DB |
| `throughput` | Total thumbnails decoded / total wall time in seconds |
| `db round-trip check` | Sanity: first thumbnail is readable back from the DB |

### Comparison with RawTherapee (M1)

Run RawTherapee on the same machine and folder. Open the folder; start a stopwatch.
Record time until the first N thumbnails appear in the Film Strip. Compare with
ferrolite's M1b number.

**Acceptance criterion:** ferrolite M1b ≤ RawTherapee time-to-first-N-thumbnails.

---

## M2: Warm Grid Scroll Smoothness

1. Run `cargo run -p ferrolite-app --release` and open the benchmark folder.
2. Wait until all thumbnails are indexed (status bar shows "Idle").
3. Scroll the grid continuously for 30 seconds; observe dropped frames visually.
4. Repeat with RawTherapee's Film Strip on the same folder.

**Acceptance criterion:** ferrolite grid scroll is perceptibly smoother (fewer
dropped frames / stutter) than RawTherapee on the same warm catalog.
